# ASCOS_ECOSYSTEM.md

AWP-004 · v1.0 · (ASCOS = program codename for the platform ecosystem, per
Program Director; rename freely — content is name-independent.)
Foundation: ABP-001 Parts A/I. This doc turns those strategies into one
ecosystem map with joining incentives per actor.

## The flywheel

Content is data (EDR-002) → partners can author it → more libraries make the
platform more valuable → more users attract more partners. Every ecosystem
program below exists to spin this wheel; anything that doesn't, we don't do.

## Customer ecosystem
- Segments & editions per COMMERCIAL.md; share-links (seed-pinned replays)
  make every customer a distribution channel (Figma loop).
- Customer council: early adopters (CUSTOMER_VALIDATION_SYSTEM) get roadmap
  voice; enterprise advisory board once ≥ 3 enterprise logos.

## OEM ecosystem (machine builders)
- **Give:** certified official machine libraries with real datasheet
  parameters ("GEA Flow Wrappers — official"), co-marketing, a pre-sales
  sim tool their reps can send as a link.
- **Get:** their machines specified into new lines; win-rate evidence.
- Mechanics: OEM Publisher agreement + certification gate (Machine Design
  Playbook checklist) + official badge. Revenue: free official libraries
  (their marketing), paid premium packs optional (70/30).

## Integrator ecosystem (system integrators)
- **Give:** project templates, FAT/SAT simulation evidence for acceptance
  tests, floating licenses for project teams.
- **Get:** de-risked proposals; billable simulation studies.
- Program: Certified Integration Partner (requires 2 certified engineers) →
  directory listing + lead routing.

## Consultant ecosystem
- The highest-leverage early channel: consultants bring the product into
  many factories on locked-down hardware (our wedge is literally their pain).
- Program: Certified Engineer status + white-label report header (their
  logo on the engineering report) at Professional tier.

## University ecosystem
- Education edition free; course kits = Academy learning paths + challenge
  projects; capstone marketplace publishing (student libraries, certified).
- Get: future engineers trained on our conventions; research citations.

## Developer ecosystem (v3+)
- Plugins = data packs + panels speaking snapshot/command protocol
  (EDR-001/006 make third-party code unable to corrupt results — the safety
  story that lets us open up at all).
- SDK = the published protocol + definition schema; registry gated by the
  same certification machinery.

## Marketplace strategy (sequenced)
1. **Now:** SKU format is frozen (MachineDefinition/template JSON) — every
   library built today is marketplace-ready.
2. **Beta:** curated catalog, manual review, free content only.
3. **v3:** payments (70/30), certified publishers, OEM official sections,
   industry packs (food → packaging → pharma → warehouse → automotive →
   semiconductor, in beachhead-adjacency order).
4. **Later:** training marketplace, calc packs, premium AI packs.

## Ecosystem KPIs
Publishers active · libraries installed/user · % of new lines using
marketplace machines · partner-sourced revenue share · certified engineers.
