1 · Overall System Architecture

A pure engineering core in the middle; everything environment-specific (rendering, storage, cloud, AI) is a replaceable adapter at the edge. New targets add adapters — the core is never redesigned. That is how one codebase covers browser, desktop, tablet, offline, cloud and AI.

Why it matters: the simulation would produce identical results with every screen deleted. Rendering watches the engine; it can never distort it. This one rule protects the engineering correctness — and the IP — permanently.

2 · Module Architecture

Dependencies point strictly downward. The two bottom packages are the long-lived intellectual property and never import from anything above them.

apps/simulator
React shell · panels · worker host · PWA · render adapters (Canvas 2D, Three.js)
▼ depends on
@dtp/project
project model · zod schema · save/load · migrations
▼
@dtp/machines
7 archetypes (source · transport · processor · buffer · splitter · combiner · sink) — real machines are data definitions, not code
▼
@dtp/engine
clock · event queue · state machines · movement · recorder — zero dependencies
side-by-side ▼
@dtp/calc
SI units · conversions · CalcValue { value, unit, formula, inputs } — every number can show its work
New machine = one definition object. New industry (pharma, beverage, warehouse) = one definition set. Zero engine changes either way.

3 · Data Flow

One-way flow; the only way anything writes into the simulation is an explicit command. A saved project plus its random seed reproduces any result exactly — simulation output becomes evidence, not animation.

USER
builds layout · sets parameters · presses Run
▼ SimCommand (start · pause · setParameter · …)
SIMULATION
deterministic kernel in a Web Worker · seeded randomness
▼ SimulationSnapshot ~30 Hz (immutable)
VISUALIZATION
2D editor and 3D view draw the same snapshot
▼ RecorderReport (time-in-state · counts · events)
DASHBOARD
OEE = A × P × Q · throughput · bottleneck · timeline — computed by the recorder, never by the UI
▼ versioned JSON (layout + parameters + seed)
STORAGE
v1: local file, fully offline · v2: cloud sync — same format, different transport

4 · Navigation Concept

One workspace, four modes sharing the same project: Design (2D layout), Simulate (3D + live line), Analyze (dashboards & reports), Library (machine definitions). Left panel and right panel adapt to the mode; the timeline is global.
